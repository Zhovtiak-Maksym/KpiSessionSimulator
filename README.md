## Призначення програми
**KPI Session Simulator** - це консольна гра типу "рогалика", яка симулює процес складання сесії в КПІ. Програма демонструє у жартівливому характері, як складаються іспити.

## Опис предметної області
Предметною областю є середовище внз. Гра відображає взаємодію між студентом та викладачем під час іспиту. Ключові елементи:
- **Іспити**: Перевірка знань з різних дисциплін (ОП, АСД, Матаналіз).

- **Викладачі**: Мають унікальний характер, складність та стратегію покарання за помилки.

- **Економіка**: Використання "токенів" для купівлі предметів, що полегшують проходження.

- **Ризик**: Можливість відрахування, перездачі або "госпіталізації" через програш у міні-іграх.

## Основні можливості системи
- **Система профілів**: Створення акаунтів зі збереженням прогресу (Passed Exams, Deaths, Tokens...) у файл типу json.

- **Вибір викладачів та предметів**: Різні сценарії іспитів з власними наборами питань.

- **Динамічна складність**: Рівень питань змінюється залежно від довжини стріку неправильних питань гравця або настрою викладача.

- **Магазин перків**: Купівля бонусів («Синє Око», «Лояльність», «Ловкі пальці» тощо) через інтерфейс **Command**.

- **Міні-ігри**: При неправильній відповіді гравець може зіграти в «Блекджек» або «Рулетку», щоб уникнути покарання.

- **Система покарань**: Реалізація різних наслідків (списання токенів, перездача, відправка на ФІОТ) через патерн **Strategy**.

## Інструкція зі збирання та запуску
Проєкт розроблений на платформі **.NET 10**.

1.  **Клонування репозиторію**:
    bash
    git clone <url>
    cd KpiSessionSimulator
 
   
2.  **Збирання проєкту**:
    bash
    dotnet build
  

3.  **Запуск програми**:
    bash
    .\bin\Debug\net10.0\KpiSessionSimulator.exe
  

4.  **Запуск тестів**:
    bash
    dotnet test
    

## Інструкція з використання
1. **Авторизація**: При запуску програми можна обрати `Login` (якщо вже маєте акаунт) або `Register` (для створення нового профілю).

2.  **KPI HUB**: Головне меню, де можна переглянути свою статистику, зайти в магазин або спробувати скласти іспит.

3.  **Чорний ринок**: Можна купити перки за токени.

4.  **Іспит**: Оберіть предмет. Відповідайте на питання.
    - Якщо не знаєте відповіді, можна застосувати куплені перки.
    - При помилці оберіть: прийняти покарання або можна спробувати відігратися в міні-іграх.

## Формат і спосіб збереження даних
Дані зберігаються у форматі **JSON**. 
- **Розташування**: Папка `Data/` у корені проєкту.

- **Profiles**: `profiles.json` - список гравців з їхніми характеристиками та особистими даними (нік, пароль).

- **Questions**: Файли `op_questions.json`, `asd_questions.json`, `matan_questions.json` - імітовані бази даних питань із варіантами відповідей та рівнем складності.

## Підготовлені файли з прикладами даних
- `op_questions.json`: Набір питань з дисципліни ОП.
- `asd_questions.json`: Набір питань з дисципліни АСД.
- `matan_questions.json`: Набір питань з дисципліни Матаналіз.
- `profiles.json`: Містить всіх зареєстрованих гравців.

## Основні модулі та компоненти
- **KpiSessionSimulator.Models**: Сутності даних (`Player`, `Question`, `PlayerStats`, `Difficulty`).

- **KpiSessionSimulator.Core**: вся важлива логіка. `ProgramProcess` керує циклом іспиту, `ExamState` відслідковує поточний стан сесії.

- **KpiSessionSimulator.Teachers**: Класи викладачів, що наслідуються від `BasicTeacher`.

- **KpiSessionSimulator.Shop**: Система магазину. Через рефлексію збирає перки, помічені атрибутом `[ShopPerks]`.

- **KpiSessionSimulator.Minigames**: Реалізація `Blackjack` та `Roulette` через інтерфейс `IMiniGame`.

- **KpiSessionSimulator.Punishments**: Набір покарань через реалізацію (`IPunishment`), які викладачі застосовують до гравця.

- **KpiSessionSimulator.Services**: Допоміжні сервіси для роботи з JSON (`JsonRepository`), завантаження питань (`QuestionsLoader`) та керування профілями (`ProfileManager`).

## Конфігурація проєкту
Конфігурація задається через константи в класі `ProgramProcess.cs`:
- `QuestionsToAnswer` (8): Кількість питань в одному іспиті.
- `QuestionsToPass` (6): Поріг для складання іспиту.
- `RouletteProbability` (40%): Шанс випадіння рулетки замість блекджеку.
- Шляхи до файлів автоматично визначаються класом `PathsMacker.cs`.
